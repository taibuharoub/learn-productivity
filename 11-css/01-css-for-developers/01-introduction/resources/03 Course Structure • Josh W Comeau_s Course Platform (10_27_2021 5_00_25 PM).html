Course Structure

This course is broken up into 10 modules.

Each module can be thought of as a mini-course. On average, it should take between 3-6 hours per module, depending on how much knowledge you already have, and how thoroughly you complete the module.

This course is intended to be done in order, but that's not a hard rule. Plenty of students have found it worthwhile to jump around and explore topics that interest them.

Most modules end in a workshop. A workshop is a real-world-inspired project. We'll apply the skills we've learned in the module to solve a larger problem.

The most-relevant CSS will be on the main pane inside a <style> tag, with additional supporting CSS tucked away in a separate tab.

To be clear: I'm not advocating for <style> tags! This is just the best way I've found to present examples in this course.

Playground controls

There are several buttons! Here's what they do:

When you click the "Save your work" button, the code in the playground is persisted in localStorage, and will be restored when you revisit the lesson.

If you wish to restore the code to its original state, you can use the "Revert code" button. This will also delete any saved work. Mouse users: you'll need to click and hold for 1 second (this is to avoid accidental clicks).

Playground source

Several students have asked if this Playground component is open-source.

Unfortunately, it isn't, but I built this version of the playground on top of this open-source library: agneym/playground. It's a solid library, and I'd encourage you to fork it as well!

Asides

The blue box explaining where my Playground came from is called an Aside.

I use this component when I have extra information. Maybe it's a bit of a tangent, or maybe it's additional detail about something.
